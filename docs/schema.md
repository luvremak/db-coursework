Представлена ER-діаграма описує реляційну базу даних для системи проектного менеджменту та трекінгу часу.

Центральною сутністю виступає **Company**, яка має зв'язки «один-до-багатьох» із таблицями **Project** (проекти) та **Employee** (співробітники). Це дозволяє розмежовувати дані різних організацій.

Сутність **Task** підпорядкована конкретному проекту та містить посилання на виконавця (Employee). Облік робочого часу реалізовано через таблицю **TimeTrackingEntry**, яка виступає сполучною ланкою між працівником і задачею, фіксуючи тривалість виконання роботи.

## Індекси

- `company.code` (UNIQUE, для швидкого пошуку компанії)
- `project.code` (UNIQUE, для швидкого пошуку проекту)
- `project.company_id` (FK, для вибірки проектів компанії)
- `employee.company_id` (FK, для вибірки співробітників компанії)
- `employee.telegram_id` (для ідентифікації користувача)
- `task.project_id` (FK, для вибірки задач проекту)
- `time_tracking_entry.task_id` (FK, для вибірки записів задачі)
- `time_tracking_entry.employee_id` (FK, для вибірки записів співробітника)
